## 🔒 資料保全鐵律（最高優先，任何變更都必須遵守）

**會員已上傳的任何資料——會員帳號、交流卡所有版本（含歷史）、專案、商機廣場合作、
快訊、互動紀錄、媒體——除非使用者在 UI 上手動刪除，否則一律不得刪除。**

具體規則：
- 修改問卷、頁面、功能時只改「定義」（questions.ts、UI、API 邏輯），**不得**寫任何刪除或重置 Supabase 資料的遷移、腳本或程式路徑。
- 題目改型別或改名時，舊答案的 key 原樣保留在 `answers` JSON 中（未知 key 無害，不做清洗）。
- 示範資料種子**只在 members 表完全為空時**執行；資料庫非空時絕不重新植入或覆蓋。
- 例外（允許的刪除）僅限兩種：
  - 使用者／會員本人在 UI 上**主動**執行的刪除（含管理員的 UI 刪除操作）
  - 範例人物（`is_demo = true`，非會員上傳資料）在真實填卡人數超過 5 人時的自動退場
- 若某個需求看似需要刪除會員資料，**先停下來向使用者確認**，不得自行執行。
